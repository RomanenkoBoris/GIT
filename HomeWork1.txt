1. Перед тем как сделать локальную копию удаленного репозитория необходимо убедиться что имеется связь
между нашим ПК и платформой - удаленным хранилищем (Remote repository). Для этой связи используется
специальный SSH криптопротокол. Команда ssh -T git@github.com поможет нам ответить на этот вопрос.
При отсутствии связи, необходимо сгенерировать пару ключей командой ssh-keygen -t rsa и авторизовать себя
на гит хабе (в нашем случае) с помощью публичного ключа. Далле у интересующего нас удаленного репозитория
копируется SSH ссылка и с помощью команды clone создается локальная копия на нашем ПК.

2. Актуализация локальной копии, согласно текущего состояния удаленного репозитория, происходит с помощью команды
git pull, являющейся противоположностью команды push.

3. Команда git status воводит в консоль состояние проекта сквозь призму гит, т.е. гит анализирует наличие изменеий
у проиндексированных файлов и наличие новых неотслеживаемых файлов и папок.

С помощью команды git log можно ознакомиться с коммитами произведенныйми на текущей ветке "сверху вниз", с переходом на
родительскую ветку, вплоть до первого коммита проекта. Из этой информации можно извлечь хэш-код коммита (мин. 4 цифры),
для дальнейшего более подробного ознакомления с его содержанием.

Команда git remote -v ответит на вопрос есть ли связь между локальным и удаленными репозиториями, также нам станет
известно какой именно удаленный репозиторий используется в проекте.
